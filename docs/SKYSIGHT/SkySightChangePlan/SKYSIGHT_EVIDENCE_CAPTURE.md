# SKYSIGHT_EVIDENCE_CAPTURE.md (Short)

Goal: capture **redacted, real** HTTP responses so the SkySight adapter can be implemented without guessing.

## Where the files come from
They are generated by **your local curl commands** and saved into the repo.

Create the folder:
```bash
mkdir -p docs/integrations/skysight/evidence
cd docs/integrations/skysight/evidence
```

## Required outputs (commit redacted versions)
- `auth_success.txt` (headers + body)
- `auth_401.txt` (headers + body)
- `parameters_success.json` (body)
- `tile_template_success.json` (body)
- `legend_success.json` (body)
- `value_success.json` (body)
- `tile_sample_headers_ok.txt` (HEAD response with auth)
- `tile_sample_headers_fail.txt` (HEAD response without auth)
- `RATE_LIMIT_NOTES.md` (hand-written after observing behavior)
